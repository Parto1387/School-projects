# 🎮 School Game Projects

This repository is a collection of small Python projects I created while learning programming in school.
They’re not meant to be large-scale or professional applications, but rather early experiments where I explored game logic, UI design, and basic interactivity.

Each project reflects a stage in my learning process and how I gradually became more comfortable working with libraries and structuring code.

# 🧊🔥 Frost & Flame Tic-Tac-Toe
A themed version of the classic Tic-Tac-Toe game built with Tkinter and enhanced with sound effects.

✨ Features
Custom characters: Pyron (🔥) vs Cryon (❄️)
Image-based board instead of plain text
Sound effects using Pygame
Turn-based system with dynamic status updates
Win detection with highlighted combinations
Restart functionality
🔮 Future Improvements
Planned update: dragon-themed animations to enhance visual experience and make gameplay more dynamic

# 🎯 Playful Cuby
An arcade-style game inspired by Flappy Bird, combined with a simple quiz system that acts as a progression gate.

✨ Features
Multi-stage flow using Tkinter (intro screens + quiz)
Quiz system with score calculation
Game unlock condition based on quiz performance
Pygame-based gameplay with:
Gravity and movement mechanics
Obstacle (pipe) generation
Collision detection
Score tracking and high score system
Sound effects for actions and outcomes
Win/lose screens

🧠 Concept
This project mixes learning + gaming, where the player must pass a quiz before accessing the main game.

# 💀 Hangman
A classic word guessing game built with a graphical interface.

✨ Features
Random word selection
Interactive letter buttons (A–Z)
Visual hangman progression using images
Real-time word display updates
Win/lose detection
Restart system
Feedback messages for correct/incorrect guesses

# 🛠️ Technologies Used
Python
Tkinter (GUI)
Pygame (sound & game loop)
PIL / Pillow (image processing)

# ⚠️ Note
These projects were created as early school work, so they are intentionally simple and not optimized for production use.
The focus was on learning fundamentals like:
Structuring code
Working with libraries
Building interactive programs

# 🚀 Why This Repo Exists
Instead of aiming for perfection, this repository shows progress.
It represents how I started, experimented, and improved over time — which matters more than building something overly complex too early.
